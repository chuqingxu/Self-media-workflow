# Self-media Workflow

这是一个面向个人自媒体和私域转化的工作流仓库。当前重点是一个 Codex skill：

- `skills/ai-zhuangjishi-shortvideo/`: **AI 装机师短视频工作流**

这个仓库不是普通文案模板库，也不是一键自动发视频工具。它的作用是把一个清晰的人设、平台打法、脚本结构、拍摄要求、剪辑说明和私域承接方式，固化成 Codex 可以反复调用的工作流。

## 当前 Skill

### AI 装机师短视频

定位：

- 人设：AI 装机师
- 一句话介绍：以前帮人装电脑系统，现在帮老板装公司 AI 系统。
- 目标人群：二三线城市中小企业老板、小团队老板、传统行业老板
- 风格：老板朋友型，真人口播优先
- 平台：抖音 + 微信视频号
- 引流：个人微信
- 承接：中转站 token / AI 使用入口，企业定制 AI 项目

核心承诺：

```text
帮二三线中小企业老板，把 AI 从“听说很厉害”装成“公司每天能用”。
```

这个人设不做“泛 AI 博主”，也不主打宏大趋势。它更像十多年前帮人装电脑系统的人：老板不需要理解所有底层技术，只需要有人帮他把入口、账号、资料、权限、流程和员工使用方式配置好。

## 工作原理

这个 skill 的工作原理可以理解为一条短视频生产线：

```text
输入主题
  -> 套入 AI 装机师人设
  -> 判断老板真实痛点
  -> 拆成抖音版和视频号版
  -> 生成真人口播脚本
  -> 补齐拍摄镜头和剪辑说明
  -> 配套标题、封面字、话题、评论区引导
  -> 给出加微信后的承接话术
  -> 判断更适合卖 token、推企业定制项目，还是做信任沉淀
```

它不是简单地“写一段短视频文案”。它会同时考虑三件事：

1. **账号人设是否统一**
   每条内容都要听起来像“AI 装机师”，而不是今天像工具测评，明天像知识博主，后天像软件销售。

2. **平台逻辑是否区分**
   抖音更看重 3 秒钩子、冲突感、完播率和高密度表达；视频号更看重可信感、老板朋友语气、真实案例和私域承接。

3. **内容是否能转化**
   公开视频里不硬推 token 和价格，而是先用检查清单、自测表、诊断问题引导加微信。加微信后再区分：低门槛 AI 使用入口，还是企业定制 AI 项目。

## Skill 负责什么

当你给一个主题时，它会默认输出：

- 主题判断：这个主题为什么适合 AI 装机师 IP
- 抖音版 30-45 秒真人口播脚本
- 视频号版 45-90 秒真人口播脚本
- 3 个开头钩子
- 拍摄镜头清单
- 剪映剪辑说明
- 标题、封面字、话题
- 评论区引导
- 加微信后的首句承接话术
- 转化判断：更适合承接 token、企业定制项目，还是信任沉淀

当你要做批量内容时，它会输出选题表：

- 日期
- 栏目
- 选题
- 目标老板痛点
- 抖音钩子
- 视频号角度
- CTA
- 更适合承接 token / 定制项目 / 信任沉淀

当你要求“落文件夹”时，它会按项目文件夹生成实际内容文件。

## 内容栏目

默认栏目有六类：

| 栏目 | 作用 | 示例方向 |
| --- | --- | --- |
| 装系统 | 讲公司 AI 基础能力怎么搭 | 公司用 AI 不是买会员，是先装入口、资料、流程 |
| 装软件 | 讲工具、模型、入口、token 怎么选 | 小老板先别买十几个 AI 工具 |
| 装驱动 | 讲员工、资料、权限、流程怎么接起来 | 员工不用 AI，往往不是懒，是没有接进工作 |
| 杀病毒 | 清理 AI 焦虑、伪需求、乱买软件 | 这 3 种 AI 项目，中小公司先别碰 |
| 上门案例 | 用匿名案例讲落地变化 | 一个门店老板怎么先把客服话术装进 AI |
| 老板问答 | 回答真实老板疑问 | 公司适不适合 AI？先花多少钱？员工不会用怎么办？ |

这些栏目是为了让账号长期稳定，而不是每天临时想一个孤立选题。

## 平台打法

### 抖音

抖音版本默认更短、更冲、更直接：

- 30-45 秒为主
- 前 3 秒必须出现老板痛点或冲突
- 不做长铺垫，不先解释背景
- 口播密度高，字幕大，节奏快
- 结尾用评论关键词、检查清单、自测表引导

适合的开头：

```text
老板，你公司不是用不了 AI，是你根本没把 AI 装进工作流程里。
```

```text
买了 AI 会员员工还是不用？问题大概率不在员工。
```

### 微信视频号

视频号版本默认更稳、更像朋友聊天：

- 45-90 秒为主
- 多讲真实场景和判断逻辑
- 少一点爆点，多一点可信感
- 适合承接个人微信和后续咨询

适合的开头：

```text
我最近发现，很多老板买 AI 会员以后，公司里还是没人真正用起来。
这个问题其实很像以前电脑刚普及的时候，只装系统不装驱动。
```

## 私域承接逻辑

公开视频的 CTA 不建议直接说“买 token”或“找我做项目”。更好的前端引导是：

- 《中小老板 AI 装机检查清单》
- 《公司 AI 基础能力自测表》
- 《员工用 AI 不乱套的 5 个配置》

用户加到个人微信后，再根据需求分流：

| 用户状态 | 适合承接 |
| --- | --- |
| 只是想低成本用 AI | 中转站 token / AI 使用入口 |
| 说公司资料、客服、销售、报价、运营流程很乱 | 企业定制 AI 项目 |
| 只是好奇，还没有明确需求 | 先发检查清单，做信任沉淀 |

默认诊断问题：

```text
你现在最想让 AI 先帮公司处理哪一块，客服、销售、文档，还是员工日常提效？
```

## 安装到 Codex

把仓库里的 skill 复制到 Codex 全局 skills 目录：

```bash
cp -R skills/ai-zhuangjishi-shortvideo ~/.codex/skills/
```

安装后，Codex 会在合适的短视频、自媒体、抖音、视频号、个人 IP 引流任务里使用它。

## 推荐目录结构

skill 本体建议全局安装，内容资产放在单独项目文件夹。

```text
~/.codex/skills/ai-zhuangjishi-shortvideo/
  SKILL.md
  agents/openai.yaml

/Users/chu/ai视频号内容/
  账号定位.md
  7天测试计划.md
  选题池/
  脚本/
  拍摄清单/
  剪辑说明/
  发布文案/
  复盘/

```

这样做的好处是：

- skill 是全局能力，之后换项目也能复用
- 视频内容文件夹只放真实产出，不混入工作流规则
- 后续可以同时支持多个账号或栏目

## 常见用法

生成单条内容：

```text
用 AI 装机师短视频 skill，围绕“老板买了 AI 会员为什么员工还是不用”生成抖音和视频号脚本。
```

生成 7 天测试计划：

```text
用 AI 装机师短视频 skill，给 AI 装机师这个 IP 做 7 天选题测试计划。
```

落到指定文件夹：

```text
用 AI 装机师短视频 skill，在 /Users/chu/ai视频号内容 里生成 7 天选题测试计划和第一条脚本。
```

改写已有素材：

```text
用 AI 装机师短视频 skill，把这段内容改成老板朋友型真人口播，分别适配抖音和视频号。
```

做复盘：

```text
用 AI 装机师短视频 skill，根据这 7 条视频的数据，判断下一周应该保留哪些栏目、砍掉哪些选题。
```

## 边界

这个 skill 不负责：

- 自动发布抖音或视频号
- 自动剪辑成片
- 保证涨粉或成交
- 替代真实拍摄和复盘
- 直接生成夸大承诺或违规广告话术

它负责的是让内容生产不再从空白开始，让账号每条内容都围绕同一套人设、目标人群、平台逻辑和成交路径。
